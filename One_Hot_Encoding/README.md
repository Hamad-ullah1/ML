One-Hot Encoding (Handling Categorical Data)

This repository contain practice work of Machine Learning course. The focus of this session is learning **One-Hot Encoding**, a widely used technique for converting nominal categorical features into numerical representations that can be used by machine learning algorithms.

## 📌 Topics Covered

- Introduction to One-Hot Encoding
- Why One-Hot Encoding is Needed
- Nominal vs Ordinal Data
- Creating Dummy Variables
- Dummy Variable Trap
- One-Hot Encoding with Pandas
- One-Hot Encoding using Scikit-learn
- Handling High Cardinality Features

## 🛠️ Technologies Used

- Python
- Pandas
- NumPy
- Scikit-learn
- Jupyter Notebook

## 📂 Project Structure

```
│── One_Hot_Encoding.ipynb
└── README.md
```
## 📖 Learning Outcomes

After completing this project, I learned how to:

- Differentiate between nominal and ordinal categorical data.
- Apply One-Hot Encoding to nominal features.
- Avoid the Dummy Variable Trap by dropping one dummy column.
- Use `get_dummies()` in Pandas for quick encoding.
- Implement `OneHotEncoder` from Scikit-learn in machine learning pipelines.
- Prepare categorical features for machine learning models.
